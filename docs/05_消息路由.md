# 第 5 章：消息路由

> 目标：理解多个 Agent 之间如何通信，以及同步路由的递归陷阱与异步解法。

## 5.1 这一章解决什么问题

单个 Agent 能思考和用工具了。但很多任务需要**多个 Agent 分工合作**——研究员搜索资料、作家写报告、审核员把关。它们之间要互相传递消息。

这就需要一个**消息路由**机制：A 发消息给 B，B 处理后发给 C。本章讲清楚这套机制，以及它隐藏的一个致命陷阱（递归栈溢出）和解决方案。

## 5.2 Message——消息协议（`core/message.py`）

Agent 间通信的基本单元：

```python
class Message:
    def __init__(self, sender, receiver, content, msg_type="text", metadata=None):
        self.id = uuid4()          # 唯一标识
        self.sender = sender       # 发送者 agent_id
        self.receiver = receiver   # 接收者 agent_id
        self.content = content     # 内容（文本/字典/任意）
        self.type = msg_type       # 类型（text/action/result 等）
        self.timestamp = now()     # 时间戳
        self.metadata = metadata   # 扩展字段
```

`content` 可以是任意类型——字符串、字典、甚至嵌套结构，由 Agent 自己约定怎么解析。

## 5.3 Router——同步消息总线（`core/router.py`）

Router 是中心化的消息总线，所有 Agent 注册到它上面：

```python
from core import Router, Message

router = Router()
router.register_agent(researcher)   # 自动绑定 researcher.router = router
router.register_agent(writer)

# researcher 给 writer 发消息
router.route_message(Message(
    sender="researcher",
    receiver="writer",
    content="这是研究资料：...",
    msg_type="research_result"
))
```

`route_message` 的内部逻辑：
```
1. 查 routing_rules（可覆盖 receiver）
2. 记入 message_history
3. 找到 receiver 对应的 Agent，调用 agent.receive(message)
```

### Agent 怎么发消息

Agent 通过 `send()` 方法发消息，它自动走 Router：

```python
# 在 Agent 的 receive() 里
def receive(self, message):
    result = self.think(message.content)
    self.send("writer", result, "research_result")   # 发给 writer
```

`send` 内部：构建 Message → 调用 `self.router.route_message(message)`。

### 异常边界

Router 的 `route_message` 用 try/except 包裹 `agent.receive()`：

```python
try:
    self.agents[receiver_id].receive(message)
except Exception as e:
    logger.exception("Agent 处理消息异常")  # 记录但不中断
    self.stats["routing_errors"] += 1
```

**一个 Agent 崩溃不会中断整个消息流**——这是工程健壮性的关键。

## 5.4 ⚠️ 同步路由的递归陷阱

这是同步 Router 最隐蔽的问题。看这个链路：

```
router.route_message(msg)            # ① 路由到 researcher
  └─ researcher.receive(msg)         # ② researcher 处理
       └─ researcher.send("writer", ...)   # ③ 内部又调 route_message
            └─ router.route_message(msg2)  # ④ 路由到 writer（同一调用栈！）
                 └─ writer.receive(msg2)   # ⑤ writer 处理
                      └─ writer.send("user", ...)  # ⑥ 又调 route_message
                           └─ ...          # ⑦ 继续嵌套
```

**每一跳都叠在同一个调用栈上**。如果链路很长，或者出现循环（A→B→A→B...），就会 **RecursionError（栈溢出）**。

这是同步"直接调用"模型的固有缺陷——`route_message` 不是把消息"投递"出去，而是"立即处理"，导致嵌套。

## 5.5 AsyncRouter——异步路由解法（`core/async_router.py`）

AsyncRouter 用 **asyncio.Queue** 把"投递"和"处理"解耦：

```
AsyncRouter.route(msg)
  └─ await queue.put(msg)     # 只入队，立即返回（不调用 receive）
                                消息处理在后台协程里串行进行
```

### 工作原理

```
         ┌──────────────────────────┐
route ──►│      asyncio.Queue       │
         │  [msg1] [msg2] [msg3]... │
         └────────────┬─────────────┘
                      │ 逐个出队
                      ▼
              ┌──────────────┐
              │ 消费协程      │ ──► agent.receive(msg)
              │ (后台运行)    │
              └──────────────┘
```

关键：Agent 的 `send` 把消息**入队就返回**（await 让出控制权），不再嵌套在调用栈里。即使 A→B→A→B 互相转发，每跳都是"入队 → 出队"的循环，**调用栈深度恒定**，永不溢出。

### 用法

```python
import asyncio
from core import AsyncRouter, Message

async def main():
    router = AsyncRouter()
    router.register_agent(agent_a)
    router.register_agent(agent_b)
    await router.start()                          # 启动后台消费协程
    await router.route(Message("user", "a", "ping"))
    await router.drain()                          # 等待所有消息处理完
    await router.stop()

asyncio.run(main())
```

### 同步兼容桥

现有 Agent 的 `send()` 是同步的、会调 `router.route_message`。AsyncRouter 提供了 `route_message` 同步桥接方法：它把消息塞进队列就返回（不阻塞、不递归），让同步 Agent 无需改造也能享受异步的好处。

## 5.6 何时用 Router vs AsyncRouter

| 场景 | 推荐 |
|------|------|
| 简单流水线（A→B→C，深度浅） | Router（同步，简单） |
| 长链路或可能循环 | AsyncRouter（防栈溢出） |
| 需要高并发处理消息 | AsyncRouter（队列天然异步） |
| 学习、原型、demo | Router（直观易懂） |

## 5.7 一个完整的多 Agent 协作示例

```python
from core import Router, Message, Agent

class Researcher(Agent):
    def receive(self, msg):
        result = self.think(msg.content)        # 思考
        self.send("writer", result)             # 发给作家

class Writer(Agent):
    def receive(self, msg):
        report = self.think(f"整理成报告：{msg.content}")
        self.send("user", report)               # 发给用户

class User(Agent):
    def receive(self, msg):
        print("最终结果:", msg.content)

router = Router()
router.register_agent(Researcher("researcher", "研究员", llm_client=llm))
router.register_agent(Writer("writer", "作家", llm_client=llm))
router.register_agent(User("user", "用户"))

router.route_message(Message("user", "researcher", "研究量子计算"))
# 流程：user→researcher→(思考)→writer→(思考)→user→(打印)
```
