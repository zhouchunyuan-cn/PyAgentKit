# 第 3 章：工具系统

> 目标：理解 Agent 如何调用外部工具，以及框架如何保证安全。

## 3.1 这一章解决什么问题

LLM 本身只会"说话"——它算不准大数乘法、查不到实时新闻、读不了你的文件。**工具（Tool）**就是给 Agent 装上"手"，让它能与外部世界交互。

但让 LLM 调用工具涉及三个难题：
1. **怎么告诉 LLM 有哪些工具可用？** —— 用 JSON Schema 描述（function-calling）
2. **怎么执行 LLM 指定的工具调用？** —— ToolRegistry 按名查找执行
3. **怎么防止 LLM 干危险的事？** —— 安全解析、路径限制、只读模式

## 3.2 Tool 基类

所有工具继承 `Tool`（`core/tools.py`）：

```python
class Tool:
    def __init__(self, name, description, parameters=None):
        self.name = name           # 工具名（LLM 引用它的标识）
        self.description = description   # 描述（LLM 据此判断何时用）
        self.parameters = parameters    # JSON Schema（描述参数）

    def run(self, **kwargs):
        raise NotImplementedError   # 子类实现具体逻辑

    def to_openai_schema(self):
        """转换为 LLM 能理解的工具描述"""
        return {"type": "function", "function": {
            "name": self.name,
            "description": self.description,
            "parameters": self.parameters,
        }}
```

`parameters` 是 JSON Schema，告诉 LLM "这个工具接受什么参数"。例如 calculator：

```python
parameters = {
    "type": "object",
    "properties": {
        "expression": {"type": "string", "description": "数学表达式"}
    },
    "required": ["expression"],
}
```

## 3.3 ToolRegistry——工具的注册表

每个 Agent 有一个 `tool_registry`，管理它能用的工具：

```python
registry = ToolRegistry()
registry.register(CalculatorTool())
registry.register(WebSearchTool())

# Agent.think() 内部这样调用：
registry.to_openai_tools()            # 导出 schema 给 LLM
registry.execute("calculator", expression="2+3")  # 执行具体工具
```

## 3.4 内置工具详解

### CalculatorTool——安全计算器 ⭐

**问题**：直接用 `eval()` 执行 LLM 生成的表达式，等于把代码执行权交给了模型——`eval("__import__('os').system('rm -rf /')")` 就完蛋了。

**解决方案**：用 Python 的 `ast` 模块做安全解析，只允许数字和基本运算符：

```python
def eval_expr(self, expr):
    # 1. 字符白名单过滤
    if not re.match(r'^[0-9+\-*/().% ]+$', expr):
        raise ValueError("表达式包含不允许的字符")
    # 2. 解析成 AST，递归求值（绝不会执行任意代码）
    node = ast.parse(expr, mode='eval')
    return self._eval_node(node.body)
```

`_eval_node` 只接受三种节点：`Constant`（数字）、`BinOp`（二元运算）、`UnaryOp`（一元运算）。任何函数调用、变量名都会被拒绝。

### WebSearchTool——网络搜索

采用**双路召回**策略：

```
搜索 "Python asyncio"
   │
   ├─ 路径1: Instant Answer API（结构化摘要）
   │   └─ 调 api.duckduckgo.com，有摘要就用
   │
   └─ 路径2: HTML 解析（多结果兜底）
       └─ 解析 lite.duckduckgo.com 的结果链接
       └─ 对中文等无 Instant Answer 的查询，这里能拿到多条网页结果
```

返回结构化结果：`{"results": [{"title": ..., "snippet": ..., "url": ...}]}`。

### FileReadTool——文件读取（带路径安全）

防止 LLM 用 `../../etc/passwd` 越界读取：

```python
def run(self, path):
    real_path = os.path.realpath(path)   # 解析真实路径（消除 ..）
    allowed = os.path.realpath(self.allowed_dir)
    if not real_path.startswith(allowed + os.sep):
        raise ValueError("不允许访问该路径（超出工作目录）")
    # 只有在允许目录内才读取
```

### DatabaseTool——真实数据库

基于 sqlite3（Python 标准库），自动区分查询和执行：

```python
db = DatabaseTool(":memory:")       # 内存库
db.run("CREATE TABLE users (id INTEGER, name TEXT)")
db.run("INSERT INTO users VALUES (1, 'Alice')")
result = db.run("SELECT * FROM users")
# → {"rows": [{"id":1, "name":"Alice"}], "row_count": 1}

db_readonly = DatabaseTool(":memory:", readonly=True)
db_readonly.run("DELETE FROM users")  # → {"error": "只读模式不允许..."}
```

## 3.5 HttpTool——网络工具基类

需要联网的工具（WebSearch 等）继承 `HttpTool`，复用统一的**超时 + 指数退避重试**：

```python
class HttpTool(Tool):
    def _http_get(self, url, headers=None):
        # 带 timeout，失败时对网络/限流类错误（429/5xx）重试
        for attempt in range(self.max_retries + 1):
            try:
                return urlopen(req, timeout=self.timeout).read().decode()
            except Exception as e:
                if attempt < self.max_retries and self._is_retryable(e):
                    time.sleep(self.retry_base_delay * (2 ** attempt))
                    continue
                raise
```

## 3.6 自己写一个工具

```python
from core.tools import Tool

class TimeTool(Tool):
    def __init__(self):
        super().__init__(
            name="current_time",
            description="获取当前时间",
            parameters={"type": "object", "properties": {}}  # 无参数
        )

    def run(self):
        from datetime import datetime
        return {"time": datetime.now().strftime("%Y-%m-%d %H:%M:%S")}

# 注册到 Agent
agent.tool_registry.register(TimeTool())
```

LLM 现在就能通过 `current_time` 工具获取实时时间了。

## 3.7 安全设计总结

| 风险 | 防护机制 |
|------|----------|
| 任意代码执行（eval 注入） | CalculatorTool 用 AST 白名单解析 |
| 文件越界读取 | FileReadTool 用 realpath + 目录白名单 |
| 数据库误删 | DatabaseTool 的 readonly 模式 |
| 网络请求挂死 | HttpTool 的 timeout |
| 工具执行崩溃 | Agent 的 `_execute_tool_safe` 异常边界 |

## 3.8 与其他模块的关系

- `Agent.think()` 通过 `tool_registry.execute()` 调用工具（第 2 章）
- `ToolFactory` 按名称实例化工具，供 YAML 配置使用（第 8 章）
- `to_openai_schema()` 产出 LLM function-calling 所需的格式
