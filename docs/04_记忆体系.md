# 第 4 章：记忆体系

> 目标：理解 Agent 如何"记住"过去，以及三类记忆如何统一为一个接口。

## 4.1 这一章解决什么问题

人对话时会记得上文——"我叫小明"之后，你问"我叫什么"我能答出来。但 LLM 本身是无状态的，每次调用都是全新的。

要让 Agent 有"记忆"，需要在调用 LLM 前，把相关信息塞进 messages。但信息从哪来、怎么组织、怎么取舍？这就是记忆系统要解决的问题。

PyAgentKit 有**三类记忆**，由 `MemoryManager` 统一管理：

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **KV 记忆** | 按 key 精确存取，短期有 TTL，长期持久化 | 存"用户名=小明"这类事实 |
| **向量记忆** | 按语义相关性召回 | "想起和当前任务相关的历史经验" |
| **会话历史** | 多轮对话的上下文 | "刚才用户说了什么" |

## 4.2 Memory——KV 记忆（`core/memory.py`）

最基础的记忆形式，像字典一样按 key 存取，但分两层：

```
短期记忆（short_term_memory）
  ├─ 存在内存，带时间戳
  ├─ 超过 short_term_duration（默认 1 小时）自动过期
  └─ 不持久化（重启丢失）

长期记忆（long_term_memory）
  ├─ 存在内存 + 持久化到 JSON 文件
  └─ 永不过期，重启后仍在
```

```python
from core import Memory

mem = Memory(short_term_duration=3600, persistent_storage_path="mem.json")
mem.store("user_name", "小明", memory_type="short")   # 临时
mem.store("birthday", "2000-01-01", memory_type="long")  # 永久

mem.retrieve("user_name")   # → "小明"（未过期时）
mem.retrieve("birthday")    # → "2000-01-01"（即使重启）
```

**过期机制**：`retrieve` 时检查时间戳，过期就删除并返回 None。`cleanup_expired_memory()` 主动清理所有过期项。

## 4.3 VectorMemory——向量记忆（`core/vector_memory.py`）

KV 记忆只能按精确 key 取值。但很多时候我们想"按意思"召回——比如用户问"Python 的特点"，应该能想起"Python 是解释型语言"这条经验，即使没存这个 key。

**原理**：把每条记忆转成向量，查询时计算余弦相似度，返回最相关的几条。

```python
from core import VectorMemory, LocalTfidfEmbedder

vm = VectorMemory(embedder=LocalTfidfEmbedder())   # 本地向量化，零成本
vm.add("Python 是一门解释型编程语言")
vm.add("Git 是版本控制工具")
vm.add("今天午餐吃了红烧肉")

# 语义召回
vm.search("编程语言", top_k=1)
# → [{"text": "Python 是一门解释型编程语言", "similarity": 0.73}]
```

### Embedder——向量化器（可替换）

向量化由 `Embedder` 完成，是个可注入的协议：

```python
class Embedder(Protocol):
    def embed(self, text: str) -> List[float]: ...
```

两种实现：
- **LocalTfidfEmbedder**（默认）：基于 sklearn TF-IDF，纯本地零成本。但只能识别"词项重叠"，不懂数学/同义。
- **GLMEmbedder**：用智谱 embedding-3 模型，语义理解强，但需消耗 API 额度。

### 为什么 add() 要重算所有向量？

`LocalTfidfEmbedder` 的向量空间依赖整个语料的词频统计（IDF 权重）。新增一条文本会改变词表，导致已有向量的维度/权重不一致——相似度会失真。

所以 `add()` 后会调用 `_recompute_all_vectors()`，在完整语料上重新拟合，重算每条记忆的向量。这是个 `O(n)` 操作，但对中小规模记忆库够用。

## 4.4 MemoryManager——统一记忆外观（⭐ 核心）

三套记忆各有各的接口，用起来认知负担大。`MemoryManager`（`core/memory_manager.py`）把它们组合成**一个统一接口**：

```python
class MemoryManager:
    def __init__(self, memory=None, vector_memory=None, conversation_max_history=10):
        self.memory = memory or Memory()          # KV 后端
        self.vector_memory = vector_memory          # 向量后端（可选）
        self._conversation = []                     # 会话历史
```

### 三个通道

```python
brain = MemoryManager(vector_memory=VectorMemory(embedder=LocalTfidfEmbedder()))

# 通道1: KV（委托 Memory）
brain.remember("user_name", "小明")
brain.recall("user_name")   # → "小明"

# 通道2: 向量（委托 VectorMemory）
brain.learn("Python 是解释型语言")           # 存入语义记忆
brain.recall_relevant("编程语言", top_k=3)   # 语义召回

# 通道3: 会话
brain.record_turn("user", "你好")
brain.record_turn("assistant", "你好！")
brain.get_conversation()   # → [{"role":"user",...}, {"role":"assistant",...}]
```

### build_context()——聚合三者 ⭐

这是 MemoryManager 最重要的方法。它一次性把三类记忆聚合成 LLM 能用的 messages：

```python
def build_context(self, user_input, system_prompt=None, extra_context=None):
    messages = []
    # 1. 角色设定
    if system_prompt: messages.append({"role":"system", "content": system_prompt})
    # 2. 长期记忆
    long_term = self.memory.list_long_term_memory()
    if long_term: messages.append({"role":"system", "content": f"长期记忆：\n{...}"})
    # 3. 向量召回（按 user_input 语义找相关经验）
    related = self.recall_relevant(user_input, top_k=3)
    if related: messages.append({"role":"system", "content": f"相关经验：\n{...}"})
    # 4. 会话历史
    messages.extend(self.get_conversation())
    # 5. 额外上下文
    messages.extend(extra_context or [])
    # 6. 当前输入（永远在最后）
    messages.append({"role":"user", "content": user_input})
    return messages
```

Agent 不再直接碰三套系统，只调一行：
```python
messages = self.brain.build_context(user_input, system_prompt=self.system_prompt)
```

### 优雅降级

后端都是可选的：
- 没配置 `vector_memory` → `learn` 静默跳过，`recall_relevant` 返回空列表
- 没传 `memory` → 自动创建默认 Memory
- `build_context` 在任何配置下都能正常产出 messages

## 4.5 Agent 怎么用记忆

每个 Agent 内置一个 `self.brain`（MemoryManager）：

```python
# think() 内部自动用 brain.build_context 组装上下文
agent.think("我叫小明")          # 这轮的对话被 brain 记下

# 下一轮无需任何额外操作，brain 已记得上文
agent.think("我叫什么？")        # → "你叫小明"
```

`think()` 每次结束时会调用 `_remember_interaction`，把这轮的 user/assistant 对话存进 brain 的会话通道，下次自动带上。

## 4.6 设计权衡：为什么用外观而非统一基类

三套记忆的数据模型差异大（KV 字典 / 向量矩阵 / messages 列表），强行抽象成统一基类会导致接口臃肿、要重写三个类。

外观层（MemoryManager）组合它们、对外提供统一 API，既达成"Agent 只面对一个接口"的目标，又零破坏现有实现——三个后端类完全不变。

## 4.7 与其他模块的关系

- `Agent.brain` 就是 MemoryManager（`agent.py` 的 `__init__`）
- `Agent.think()` 通过 `brain.build_context()` 组装上下文（第 2 章）
- 向量记忆的 Embedder 可切换（本地 TF-IDF 或 GLM embedding）
